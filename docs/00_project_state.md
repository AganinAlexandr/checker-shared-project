# Состояние проекта

Дата инициализации: 2026-06-13.

## Цель

Checker проверяет загружаемый PDF-раздел проектной документации на соответствие требуемым заголовкам и маркерам пунктов по ПП87 для заданного вида работ, объекта и типа раздела, а также формирует замечания по отсутствующим или некорректным пунктам.

## Текущий стратегический подход

1. Сначала определяем качество входного PDF и пригодность текстового слоя.
2. Затем ищем первый блок body: `маркер + заголовок + сразу body_text`.
3. После этого строим корпуса по зонам: TOC до body и BODY начиная с первого блока.
4. Далее выбираем правильные корпуса через структурные признаки и сравнение с ПП87.
5. Затем формируем item-level verdicts: `present`, `missing`, `wrong_text`, `wrong_marker`, `wrong_version`, `not_applicable`, `unchecked`.
