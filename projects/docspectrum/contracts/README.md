# DocSpectrum Contracts

Проектные контракты `DocSpectrum`, которые нужны и generator, и reviewer.

Примеры:

- формат review packet;
- контракт метрик сравнения;
- reason-коды для замечаний reviewer;
- правила публикации checkpoint.

Если контракт становится общим для нескольких проектов, его следует перенести или продублировать ссылкой в корневый `contracts/`.
