# DocSpectrum Eval Set Plan v0

Статус: draft
Дата: 2026-06-15

## Проблема

У `DocSpectrum` пока нет объективного ground truth для сходства разделов.

Без экспертного eval_set нельзя строго утверждать, что изменение метрики улучшило качество.

## Минимальный eval_set

Первая версия должна включать экспертные пары:

- `same_house_gvs_ot_close` - один дом, близкие разделы `ГВС` / `ОТ`;
- `same_section_same_org_close` - одинаковый тип раздела, та же организация, похожие объекты;
- `same_section_moderate_diff` - одинаковый тип раздела, похожий ряд, но заметные отличия;
- `different_section_control` - разные типы разделов как отрицательный/контрастный контроль;
- `sm_contrast_control` - `СМ` как отдельный таблично-текстовый контрольный слой.

## Минимальные поля пары

- `pair_id`
- `left_object_id`
- `left_section_code`
- `right_object_id`
- `right_section_code`
- `expected_similarity_class`
- `expected_changed_axes`
- `human_note`

## Стартовая стратегия

Текущие 8 пар `1724_25` / `1725_25` используются как smoke/evidence, но не как полноценный eval.
