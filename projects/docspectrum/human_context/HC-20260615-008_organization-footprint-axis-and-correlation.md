# HC-20260615-008: Organization / electronic-footprint axis and correlation detection

## [human]

- Переход на других проектировщиков даже для того же вида работ (УУиР) **очень сильно меняет разделы по ВСЕМ видам элементов**. Различие ВНУТРИ одной проектной организации (ПО) намного меньше, чем МЕЖДУ разными организациями — **если они не коррелированы**.
- «Коррелированы» означает конкретно:
  1. **один владелец, но разные «вывески»** (разные юр-названия одного реального субъекта);
  2. **один проектировщик может сделать свой раздел для объектов разных организаций** (общий реальный исполнитель раздела).

## [opus digest]

Это **третья ось проекта — организация / электронный почерк** (ТЗ §1, §19 organization_and_designer_signals; «геном проекта», субподрядчик из генезиса / ProjectGenome). Отлична от оси A (содержание/раздел, HC-005) и оси B (размер/ТЭП, HC-006).

- **Премиса разделимости:** d_внутри(орг) ≪ d_между(орг₁,орг₂) при некоррелированности → орги кластеризуются, почерк детектируем.
- **Корреляция = СИГНАЛ продукта, не шум**, на ДВУХ уровнях гранулярности, различаемых по паттерну across-sections:
  - **один владелец / разные вывески** → высокое сходство по БОЛЬШИНСТВУ/всем разделам между «разными» оргами → один реальный субъект (детект на уровне паспорта-агрегата объекта);
  - **общий исполнитель раздела** → высокое сходство в ОДНОМ типе раздела при низком в прочих → субподрядчик/общий проектировщик этого раздела = «смешанный геном» (детект на уровне per-section).
  - различающий паттерн: *все разделы высоко* = владелец; *один раздел высоко, прочие низко* = общий исполнитель. Требует метрики на обоих уровнях (паспорт + по-раздельно) — DocSpectrum уже производит per-section, агрегат добавляется.
- **Калибровка:** текущий внутри-РСПК корпус = САМЫЙ ТРУДНЫЙ режим (малая вариация, тонкая дискриминация, борьба с boilerplate-floor) → меж-орг разделение будет ЛЕГЧЕ (большой разрыв); калибровка на внутри-орг **консервативна**. По-настоящему трудный И ценный случай — детект КОРРЕЛИРОВАННЫХ оргов (заимствование/общий исполнитель), он ловится ровно той тонкой дискриминацией, что мы строим → текущая работа прямо служит головной цели.
- **Рекалибровка неизбежна:** пороги (near/far ТЭП, полосы сходства) откалиброваны на внутри-орг масштаб; меж-орг «far» гораздо дальше → не зашивать внутри-орг калибровку как универсальную (конкретизация HC-007). Смену масштаба при cross-org НЕ трактовать как провал метрики.
- **Cross-org eval (будущее, при наличии другого орга):** новый контроль `cross_org_same_work` (УУиР) → ожидаемо НИЗКое сходство; + **инверсионный тест**: высокое cross-org сходство → кандидат корреляции; классифицировать по across-section паттерну (все разделы = вывеска/владелец; один раздел = общий исполнитель).
