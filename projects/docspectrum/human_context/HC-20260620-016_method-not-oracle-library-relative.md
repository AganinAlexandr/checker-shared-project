## [human] — позиционирование

Мы НЕ предлагаем приложение, которое сразу скажет «раздел на X% заимствован, на Y% самостоятельный» (приложения пока нет — только алгоритмы). Мы предлагаем МЕТОД:
1. **Сначала создать БИБЛИОТЕКУ** для сравнения — без основы ничего не работает;
2. имея библиотеку — проводить сравнение, но оно работает **ТОЛЬКО В РАМКАХ библиотеки**.

## [opus digest]

- **Метод, не оракул; результат ЗАКРЫТО-МИРОВОЙ (относительно библиотеки):**
  - «заимствовано» = «совпадает с чем-то В библиотеке»; «самостоятельно» = «нет аналога в библиотеке» — НЕ абсолютная оригинальность. Заимствованное ИЗВНЕ библиотеки читается как «библиотечно-оригинальное».
  - **Честная формулировка вывода:** «совпадает с N разделами библиотеки / нет аналога на текущем покрытии», НЕ «X% заимствовано».
  - **Чувствительность = ПОКРЫТИЕ библиотеки** (отсюда п.1 «без основы не работает», п.2 «только в рамках»).
- **Два продукта-слоя (что мы и строим):** (1) методология ПОСТРОЕНИЯ библиотеки (извлечение/чистка/alias/provenance/структура почерка — бóльшая часть RP: распаковка→титулы→provenance→577-корпус); (2) внутрибиблиотечное сравнение/атрибуция (H1/H2, near-match, owner-кластеры, флаг фальсификации ИУЛ).
- **Библиотека = главный актив** (не «волшебная кнопка»); ценность растёт с курированием/покрытием. 577-корпус + слои = рабочий ЭКЗЕМПЛЯР метода.
- **Дисциплина:** держит ТЗ §5.2 (informing-not-punishing) — выводы относительные/калиброванные, без over-claim. Влияет на формулировки отчётов/пакетов (Codex): «относительно библиотеки», не абсолютные вердикты.
