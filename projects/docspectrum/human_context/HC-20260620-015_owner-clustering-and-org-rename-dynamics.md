## [human]

- **Один владелец может держать до 4 организаций** из имеющихся ~50-60; вычисляются по **схожему ОФОРМЛЕНИЮ ПД (структура+текст)** + общий ГИП.
- **Caveat:** внутри одной орг могут быть 2+ КОНКУРИРУЮЩИХ за статус групп, которые НЕ обмениваются результатами/шаблонами → орг не всегда монолитен по почерку (внутри-орг гетерогенность ожидаема).
- **Дрифт названий подтверждён на практике** (владельцы признавались). За 2 года лишь немногие орг сохранили имя; некоторые переименовались; от некоторых нет объектов → сырое расширение объектов слабо растит базу сравнений.
- **Метод (предложен owner):** временной график появления объектов на экспертизу (файл `Капремонт_Объекты`, дата_вх) → искать **одновременное угасание одного имени ↔ расцвет другого** (= rename). Связки: тот же ГИП ИЛИ оформление ПД (структура/текст).

## [opus digest] — это ГЛАВНЫЙ UC (электронный почерк → владельцы)

- **UC: кластеризация орг → ВЛАДЕЛЬЦЫ/группы** по триаде связок: (1) **почерк оформления** (структура+текст — наш валидированный H1/H2-сигнал), (2) **общий ГИП**, (3) **временной паттерн**.
- **Два детектируемых явления (подтверждены временным графиком 20-06, дата_вх поквартально):**
  - **RENAME/дрифт** = временная АНТИ-корреляция (↓↔↑) + общий ГИП + continuity почерка. Кандидаты: **Комтех→АО ССУ№3** (ГИП питанов), **СК Вектор→СтройМонтаж СП** (ГИП рузаев).
  - **Мульти-орг одновременно (один владелец/субподряд)** = КОНКУРЕНТНОЕ существование + общий ГИП/почерк. Пример: Сергеев (Ватага/Гамма/Стройинвест, все 2025).
- **Петля:** валидированный почерк (RP-030..038) ИСПОЛЬЗУЕТСЯ как связка для разрешения идентичности орг → продукт, а не только валидация.
- **Следствие для базы сравнений:** мержить rename-пары (вернуть same-org baseline) + использовать мобильность ГИП → растит H2 из ИМЕЮЩИХСЯ данных сильнее, чем сырое +70-80% объектов.
- **Уровень кластеризации:** из-за внутри-орг конкурирующих групп — кластеризовать на под-орг уровне (group), не предполагать 1 орг = 1 почерк.
