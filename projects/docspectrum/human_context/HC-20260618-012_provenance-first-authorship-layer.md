## [opus digest]

- **overlap ≠ копирование.** Даже structure 0.99 / text 0.76 (manufacturer-формы) = non-copy. Структурное сходство + редкий общий текст + сильный text-Jaccard **недостаточны** для вердикта заимствования. Провенанс первичен (ТЗ §5.2 «информируем, не караем»; correlation≠proof).
- **Общий редкий текст here = third-party-авторства** (производитель / регламент / сметный софт / вендор) — НИ ОДИН из проектных орг его не авторил → межорг-заимствованием он быть не может по определению.
- **Архитектурный сдвиг — PROVENANCE-FIRST:** до интерпретации почерка/заимствования нужен **явный слой классификации авторства/источника** контента: org-authored vs third-party-authored (manufacturer / regulatory / software-generated / vendor). 
- **Уточняет HC-009 (иерархия почерка L0-L3):** орг/ГИП/исполнитель-почерк (L1/L2/L3) живёт ТОЛЬКО в **орг-авторском ОСТАТКЕ** — после вычитания third-party-контента. Заимствование детектируется только в орг-авторском остатке между орг.
- **Дискриминатор `borrowing_candidate`:** орг-авторский ОТЛИЧИТЕЛЬНЫЙ контент, разделяемый кросс-орг БЕЗ внешнего источника. `shared_technical_content` = стандартный вендор/отраслевой материал (не отличительный). Ось = авторство + отличительность.
- **Метрик-инверсия:** наибольший overlap = наиболее стандартизованная форма (normative 0.9941) = НАИМЕНЕЕ вероятно копия. «Топ-overlap = топ-подозрение» — ложно; искать в средней полосе.
- **Негативный контроль закрыт; позитивный — GATING для UC3-порогов.** RSPK↔NK = чистый негативный контроль (0 заимствований, все объяснены). Без позитивного контроля (известный same-dev-ГИП кросс-орг / до-после экспертизы / известный реюз; нужны не-УУиР выгрузки — ждут данных) порог заимствования неоткалибруем.
