## [opus digest]

Как scope/инвариант для следующего пакета (Codex generator):

- **Типичность/самостоятельность = корпусная частотность (DF/IDF), смысл НЕ нужен.** Фрагмент частый по корпусу → шаблон/взято; уникальный → самостоятельное. Это ТЗ §3 + схема `comparison_result_model` §16 `typicality_assessment` / §17 `originality_assessment`.
- **Форма вычисления — раздел-против-БИБЛИОТЕКИ, не парная** (в отличие от v0.2 A↔B): корпусный DF-индекс строится один раз по корпусу, затем каждый раздел скорится против него. `comparison_type = section_to_library`.
- **Две библиотечные структуры под два семейства задач:**
  - *библиотека разделов* (паспорта целиком) → ближайший аналог / similarity / `source_matches` (грубое соответствие приемлемо);
  - *библиотека экстрагированных сущностей* (инвертированный индекс фрагмент→документы + DF; фрагменты = текст-сегменты/шинглы/структурные сигнатуры/ТГЭ) → типичность/самостоятельность/заимствование. `element_base_v0` — её прото-версия. Глоссарий уже подразумевает («источник элементной базы», «накопитель ТГЭ»), но как отдельную структуру её стоит назвать явно.
- **Частотность даёт 3 естественные корзины** (богаче, чем 2), одно вычисление кормит несколько блоков схемы: высокий DF = шаблон (`typicality`); низкий DF, общий с конкретными разделами = вероятное заимствование (`source_matches`/`fragment_matches`); DF=1 = самостоятельное (`originality`).
- **Инварианты ревью:** корпусный DF доменно-нейтрален (ядро, не профиль → `CORE_DOMAIN_LEAK` чист); hash-only сохраняется (работает на уже имеющихся хэшах); семантика отложена как pluggable-подслой; порог DF «типичное vs редкое» = accepted-for-experiment, калибровка после eval_set.
