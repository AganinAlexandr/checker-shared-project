## [human]

- В параллельном проекте Checker достигнута **100% идентификация титульных страниц** на тестовых разделах (кроме битых; на датасете 100%). Результат, сделанный в одном месте, переиспользуем в других, если дёшево — здесь приемлемо (цена для тестирования невелика).
- **В разделах ПД структура элементов на титульных страницах заметно отличается от иных страниц раздела, А след ИСПОЛНИТЕЛЯ на титульных ГОРАЗДО МЕНЬШЕ, чем на остальных страницах.**
- → можно сравнивать титульные страницы разных разделов, разных ГИПов, разных проектных организаций; определяются они со 100% точностью (high-confidence).
- Артефакт: `registries/title_pages_by_crc.csv` (Checker title-detector `title_zone.py`, crc32 → титульные + confidence, 451 bundle, high 443 / low 8, делимитер `;`, джойн по crc32 DEC-006).
- Главное — хорошие тесты, помогающие отлаживать алгоритмы; варианты исследований не ограничены логически-подобранными группами (35/34).

## [opus digest]

- **Титульные = ЧИСТЫЙ зонд ОРГ/ГИП-почерка (L1/L2 из HC-009).** Малый L3-шум (индивидуальный исполнитель) → элемент-почерк титульных доминирован орг/ГИП-шаблонами. Плюс **контролируемая поверхность сравнения**: один и тот же «вид» страницы по всем разделам/оргам/ГИПам → убирает конфаунд содержания раздела, изолирует орг/ГИП-стиль. (Граница: титульные НЕ зондируют L3 — это территория body-страниц.)
- Систематизирует находку RP-015 (титулы РСПК=рамки+таблицы vs НК=рамки+изображения).
- **Бонус к надёжности org-distinctiveness:** org-distinctiveness/foreign сейчас артефакт-склонен (RP-019: ratio-delta на полно-секционных формах). Почерк на титульных (низкий индивид-шум, контролируемая поверхность) — **кандидат на БОЛЕЕ НАДЁЖНУЮ основу орг-атрибуции** → потенциально чинит foreign→high reliability.
- **Scaffolding (HC-007):** title-cross-test = чистая валидационная поверхность для орг/ГИП-почерка, не ядро; confidence=high only (443), low (8) диагностика, битые исключать.

**Покрытие (проверил джойном по crc32):** Checker-registry покрывает **RSPK 279/279 (high)**, NK 0/196. NK закрыл сам (`title_zone.py` локально недоступен — на этой машине только данные, не код Checker): воспроизвёл **anchor-рецепт** (регекс `^\W*проектная\s+документация\b`, группировка text_segments в строки по (page, round(y1)), is_title_line: не «состав»/«ведомост», len≤32, без «», page≤8). Прогон по 196 NK → **190 high, 6 low**. Артефакт: `E:/output/DocSpectrum/title_pages_nk_anchor_approx_v0.csv` (source=docspectrum_anchor_approx_v0; RSPK-registry НЕ тронут).
- **6 low — ВСЕ NK СМ** (1851-1855, 1864): page 1 = летерхед-текст (орг/адрес/ИНН) + **титул растром** (1 image-элемент) → текст-якорь не ловит (image_title-случай). Не отсутствие титула. → нужен **растровый сигнал** (Checker: «крупный image-элемент на странице», точнее) ИЛИ канонический прогон Checker `title_zone.py` (с raster-fallback) по NK → дополнить shared-registry каноническими NK-строками. Anchor-approx = временный анблок.
- **Итого 469/475 покрыто** (279 RSPK + 190 NK) → **кросс-ОРГ title-зонд РАЗБЛОКИРОВАН** для cross-test; 6 СМ — известный растровый gap.
