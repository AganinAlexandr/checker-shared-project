## [2026-06-14 · human]
Принцип: **заголовок-ПП87 + ПОРЯДОК первичны; маркер — вторичный/подтверждающий.**
1. BODY, заголовок без маркера: вменяемый ПП87-заголовок НЕ вылетает из блока из-за забытого маркера — держится на форматировании (bold/CAPS/Аа) + порядке в ряду ПП87. Безмаркерный ПЛОХОЙ текст → «отсутствие пункта по ПП87» (missing).
2. BODY без маркеров: плохие заголовки → «отсутствие пунктов»; ПП87-заголовки В ПРАВИЛЬНОМ ПОРЯДКЕ → основа для «пропущенные маркеры».
3. TOC (таблица/список): искать ПП87-заголовки прямо в сыром тексте; наличие/порядок задаёт повторяющаяся структура. МОЖНО вообще без маркеров, если заголовки подходят по ПП87 и лучшего кандидата нет. Небуквенный маркер — только подтверждение (совпадение с body). Заголовки не-ПП87 без буквенных маркеров → возможно содержание неправильное/отсутствует (по СПДС).
4. Корпуса: рассмотренные = лучшие.
Уточнение: verdict_for НЕ менять (остаётся марке-центричным); безмаркерные хорошие заголовки — 2-й проход (окно ПП87-заголовков по toc-зоне), не переделка логики. Порядок соблюдается почти везде; редкая инверсия (ООС: 3 маркера, 2-й составной, обратный порядок) — НЕ жёсткий контра-сигнал.
