# SPDS Frame Text Extraction Contract

Статус: draft
Дата: 2026-06-13

## 1. Назначение

Этот контракт фиксирует отдельную задачу: надежно получить текстовую область внутри СПДС-рамки страницы.

Важно: наличие `text_segments`, `frames`, `tables` и `page_kind` в `pdf-structure-explorer` не означает автоматически, что задача извлечения текста внутри СПДС-рамки решена для Checker.

## 2. Что считается результатом

Для каждой страницы раздела должен быть получен `spds_text_region`:

- `document_id`
- `file_crc32`
- `page_number`
- `spds_frame_id`
- `frame_x1`
- `frame_y1`
- `frame_x2`
- `frame_y2`
- `inner_x1`
- `inner_y1`
- `inner_x2`
- `inner_y2`
- `frame_detection_method`
- `frame_confidence`
- `included_text_segment_ids`
- `excluded_text_segment_ids`
- `excluded_table_ids`
- `excluded_stamp_segment_ids`
- `spds_text`
- `quality_flags`
- `notes`

## 3. Базовые требования

1. СПДС-рамка должна определяться как геометрическая область страницы, а не как таблица.
2. Нижние/боковые штампы и прилипшие служебные ячейки не должны попадать в основной `spds_text`.
3. Таблицы внутри рамки сохраняются как отдельные структуры, но их текст не должен автоматически смешиваться с body-текстом.
4. Текст вне СПДС-рамки не должен попадать в область поиска первого блока и корпусов, кроме случаев явно разрешенного fallback.
5. Для титульных страниц/обложек результат может быть помечен как `title_page`, но это не должно считаться ошибкой.
6. Для графических листов результат может быть пустым или ограниченным, если `page_kind` / структура страницы указывает на графическую часть.

## 4. Источники-кандидаты

Возможные источники:

- `pdf-structure-explorer/elements.csv` с `group_id=frames`;
- `pdf-structure-explorer/text_segments.csv`;
- `pdf-structure-explorer/tables.csv` и `table_cells.csv`;
- ранее подготовленные Checker text-layer / SPDS-inner-frame extracts;
- ручные `text_layer.txt` для отдельных контрольных разделов.

## 5. Минимальный эксперимент проверки

Нужно сравнить как минимум 3 источника на одном наборе страниц:

1. Старые/ручные извлечения текста внутри СПДС-рамок, если есть.
2. Текстовые сегменты `pdf-structure-explorer`, обрезанные по выбранной рамке.
3. Текущий extractor Checker / Opus, если он использует свою логику.

Минимальные контрольные разделы:

- `1480_25 АР` / `doc_fd8c057e`;
- `61-23 КР храм/лавка` / `doc_dd8a9d98` или соответствующий bundle;
- один раздел с графической частью;
- один раздел с таблично-нагруженным body;
- один раздел со scanned/text_outlined_suspect страницами, если есть.

## 6. Метрики

Черновые метрики:

- `frame_detected_rate` - доля страниц, где рамка найдена;
- `inner_text_segment_count` - число текстовых сегментов внутри inner frame;
- `outside_text_leak_count` - число сегментов вне рамки, ошибочно включенных в `spds_text`;
- `stamp_leak_count` - число сегментов штампа, ошибочно включенных в основной текст;
- `table_body_mix_count` - число случаев, где таблица смешана с body-текстом без явной роли;
- `manual_diff_ratio` - отличие от ручного/старого эталона, если он есть;
- `heading_preservation_rate` - сохраняются ли известные заголовки ПП87 на страницах.

## 7. Открытые вопросы

1. Достаточно ли искать самую большую рамку на странице, или нужны правила для листов А3/А1 и вложенных рамок?
2. Как отделять СПДС-рамку от больших таблиц/ведомостей?
3. Где проходит граница между `spds_text` и текстом штампа?
4. Нужно ли хранить текст таблиц внутри рамки в отдельном поле `spds_table_text`, а не смешивать с `spds_text`?
5. Как учитывать страницы без СПДС-рамки: титулы, обложки, приложения, сканы?
